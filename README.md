# 디스코드 경험치 봇

경험치 / 도박(배팅) / 출석체크 / 서버순위 / 선물 / 관리자 지급 / 역할상점 / 가챠 / 음성·채팅 활동 통계 기능을 갖춘 디스코드 봇입니다.

## 1. 사전 준비

1. Node.js 18 이상 설치
2. https://discord.com/developers/applications 에서 애플리케이션 생성 → Bot 탭에서 봇 생성 후 토큰 발급
3. Bot 탭에서 아래 **Privileged Gateway Intents** 를 반드시 켜기
   - MESSAGE CONTENT INTENT (채팅 경험치 지급용)
   - SERVER MEMBERS INTENT (닉네임/멤버 조회용)
4. OAuth2 → URL Generator 에서 `bot`, `applications.commands` 체크, 권한(Permissions)에서 아래 항목 체크 후 생성된 링크로 서버에 초대
   - Manage Roles (역할 지급/구매를 위해 필수)
   - Send Messages, Embed Links, Attach Files, Use Slash Commands
   - **주의**: 봇의 역할이 지급하려는 역할들보다 서버 역할 목록에서 위에 있어야 합니다.

## 2. 설치

```bash
npm install
cp .env.example .env
```

`.env` 파일을 열어 `DISCORD_TOKEN`, `CLIENT_ID` 를 입력하세요. (테스트 중이라면 `GUILD_ID`도 입력하면 즉시 반영됩니다)

## 3. 한글 폰트

`/경험치` 명령어의 프로필 카드 이미지 폰트는 `card.js`에서 `fonts/` 폴더의 파일을 읽어 등록합니다. 이미 폰트가 적용되어 있다면 그대로 두면 됩니다.

## 4. 슬래시커맨드 등록 & 실행

```bash
npm run deploy   # 슬래시커맨드 등록 (명령어를 추가/수정할 때마다 1회 실행)
npm start        # 봇 실행 (배포 스크립트를 자동으로 먼저 실행합니다)
```

## 6. 역할상점 / 가챠 사용 순서

이제 순서 상관없이 사용할 수 있습니다.

- **역할상점**: `/역할상점`으로 먼저 게시해도 되고, `/역할추가`부터 해도 됩니다. `/역할추가`, `/역할제거`를 실행하면 이미 게시된 상점 메세지가 있는 경우 **자동으로 드롭다운 목록을 갱신**합니다. (임베드 본문에는 관리자가 입력한 메세지만 표시되고, 역할/가격 목록은 드롭다운 메뉴 안에서만 보입니다.)
- **가챠**: `/가챠`로 먼저 게시해도 되고, `/가챠확률설정`부터 해도 됩니다. 뽑기 버튼은 클릭하는 시점의 최신 확률을 그대로 사용하므로 별도 갱신이 필요 없습니다. (임베드 본문에는 관리자가 입력한 메세지만 표시되며, 확률/역할 목록은 공개되지 않습니다.)
- 가챠 1회 뽑기 비용은 경험치 1000입니다. (`index.js`의 `GACHA_COST_XP` 값으로 조절 가능)

## 7. 도박(배팅) 규칙

- `/도박 배팅량` 실행 시 최소 1000 도박경험치부터 배팅 가능
- 배팅 임베드의 "🎲 배팅 시작!" 버튼을 눌러야 실제로 배팅이 진행됩니다. (버튼은 배팅을 시작한 본인만 누를 수 있음)
- 배율 확률 (`index.js`의 `GAMBLE_OUTCOMES`에서 조절 가능):
  - 0배 (전액 손실): 50%
  - 1배 (본전): 30%
  - 2배: 10%
  - 3배: 7%
  - 5배: 3%
- 버튼을 누르는 즉시 결과가 확정되고, 같은 버튼은 재사용할 수 없습니다.

## 8. 경험치 / 통계 자동 지급 설정 (index.js 상단)

```js
const XP_PER_MESSAGE = 50;       // 채팅 1개당 경험치
const XP_PER_VOICE_MINUTE = 100; // 음성채널 1분당 경험치
```

채팅 횟수(`/채팅`)와 음성 활동 시간(`/음성`)은 경험치와 별개로 항상 누적 기록됩니다.

## 9. 참고 / 주의사항

- 데이터는 `DB_PATH` 환경변수(Railway 볼륨 등) 또는 프로젝트 폴더의 `data.sqlite` 파일에 저장됩니다 (서버별로 자동 분리).
- 기존에 운영 중이던 DB를 그대로 사용해도, 처음 실행 시 필요한 컬럼(`voice_seconds`, `message_count` 등)이 자동으로 추가됩니다.
- 음성 통계는 1분마다 그 시점에 음성채널에 있는(봇 제외, AFK 채널 제외) 모든 인원에게 한번에 지급/누적하는 방식입니다.
